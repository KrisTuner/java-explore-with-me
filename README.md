# 🌍 Explore With Me - Платформа для поиска событий

Java Spring Boot PostgreSQL Maven Docker

## 📋 О проекте

Explore With Me — это многомодульное приложение для поиска и организации событий, разработанное на микросервисной архитектуре. Платформа позволяет пользователям создавать события, оставлять комментарии, подавать заявки на участие, а также просматривать подборки интересных мероприятий.

Проект представляет собой аналог афиши мероприятий, где каждый пользователь может стать как участником, так и организатором.

### Архитектура
Проект состоит из двух основных сервисов:

| Сервис | Порт | Назначение |
|--------|------|------------|
| Main Service | 8080 | Основной сервис с бизнес-логикой приложения |
| Stats Service | 9090 | Сервис сбора и анализа статистики посещений |
| Stats DB | 5433 | БД статистики (PostgreSQL) |
| Main DB | 5432 | БД основного сервиса (PostgreSQL) |

### 🎯 Цели и задачи проекта
*   Создание удобной платформы для поиска и организации событий
*   Реализация микросервисной архитектуры с разделением ответственности
*   Сбор и анализ статистики для улучшения пользовательского опыта
*   Обеспечение масштабируемости и отказоустойчивости
*   Разработка системы модерации контента (событий и комментариев)

## 🔍 Функциональность

### 👤 Пользователи (User Service)
*   Регистрация и управление профилем
*   Просмотр и редактирование своих данных
*   Разделение прав (пользователь/администратор)

### 📅 События (Event Service)

#### Публичный доступ:
*   Поиск событий по различным критериям (текст, категории, даты)
*   Фильтрация по доступности и платности
*   Сортировка по дате и популярности
*   Просмотр детальной информации о событии

#### Для авторизованных пользователей:
*   Создание и редактирование своих событий
*   Управление заявками на участие
*   Просмотр статистики по своим событиям

#### Для администратора:
*   Модерация событий (публикация/отклонение)
*   Поиск событий по различным критериям
*   Редактирование любых событий

### 💬 Комментарии (Comment Service) — Новая фича!
*   Оставление комментариев к опубликованным событиям
*   Статусная модель комментариев: PENDING → PUBLISHED/REJECTED
*   Модерация комментариев администратором
*   Редактирование и удаление своих комментариев
*   Просмотр комментариев к событиям для всех пользователей

### 🏷️ Категории (Category Service)
*   Создание и управление категориями событий
*   Обязательная валидация уникальности названия
*   Защита от удаления категорий с существующими событиями

### 🎪 Подборки (Compilation Service)
*   Формирование тематических подборок событий
*   Возможность закрепления подборок (pinned)
*   Динамическое обновление статистики просмотров

### 📊 Статистика (Stats Service)
*   Сбор данных о просмотрах событий
*   Уникальные и неуникальные просмотры
*   Фильтрация по датам и URI
*   Анализ популярности мероприятий

### Предварительные требования
Убедитесь, что у вас установлены:
*   Docker (версия 24.0 или выше)
*   Docker Compose (версия 2.20 или выше)
*   Git (для клонирования репозитория)

### Пошаговая инструкция

1.  **Клонирование репозитория**
    ```bash
    git clone https://github.com/your-username/java-explore-with-me.git
    cd java-explore-with-me
    ```

2.  **Настройка переменных окружения (опционально)**
    В корне проекта есть файл `.env`. В нем можно изменить названия баз данных, пользователей и пароли. По умолчанию используются значения, указанные в этом файле.
    ```bash
    # Пример содержимого .env
    STATS_DB_NAME=stats_db
    STATS_DB_USER=stats_user
    STATS_DB_PASSWORD=stats_password
    EWM_DB_NAME=ewm_db
    EWM_DB_USER=ewm_user
    EWM_DB_PASSWORD=ewm_password
    ```

3.  **Сборка проекта**
    Соберите проект с помощью Maven (убедитесь, что Maven установлен, или используйте обертку `./mvnw`):
    ```bash
    mvn clean package
    ```

4.  **Запуск с помощью Docker Compose**
    ```bash
    docker-compose up -d
    ```

5.  **Проверка работы**
    *   Основной сервис будет доступен по адресу: `http://localhost:8080`
    *   Сервис статистики: `http://localhost:9090`

## 🧪 Запуск тестов и проверка кода

Проект использует многомодульную структуру Maven. Для запуска тестов и проверки качества кода выполните в корневой папке проекта:

```bash
# Запуск всех тестов
mvn test

# Проверка стиля кода (Checkstyle)
mvn checkstyle:check

# Анализ покрытия тестами (JaCoCo) и генерация отчета
mvn verify
# Отчет по покрытию будет в /target/site/jacoco/index.html каждого модуля
